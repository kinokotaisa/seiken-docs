## 1. 概要

### 1.1. プロジェクトの目的

本文書は、「遠隔操作用Webサイト」プロジェクトのフロントエンド（Webブラウザ上で動作するシングルページアプリケーション）の設計仕様を定義するものである。

本アプリケーションは、ユーザーがIoT機器（正拳突き人形）を直感的に管理・操作するためのGUIを提供する。ユーザー認証、登録デバイスの管理、リアルタイムな状態監視、遠隔操作、および活動データの可視化といった機能を実装する。

### 1.2. 本書の目的

本書は、本フロントエンドアプリケーションのアーキテクチャ、機能、実装詳細、セットアップ手順、および運用方法を網羅的に記述し、開発者間の共通理解を形成するとともに、将来のメンテナンスや機能拡張の際の技術的な参照資料となることを目的とする。

## 2. アーキテクチャと技術スタック

### 2.1. アーキテクチャ

本アプリケーションは、Vue.js 3のComposition APIをベースとした、コンポーネントベースのSPAとして構築される。各画面は「ビュー」コンポーネントとして定義され、「ルーター」によってURLと動的に紐付けられる。バックエンドサーバーとの通信は、REST APIとWebSocket（Socket.IO）を介して行われる。

### 2.2. 技術スタック

| 分類 | 技術・ライブラリ | 選定理由 |
| :---- | :---- | :---- |
| **フレームワーク** | Vue.js 3 (Composition API) | リアクティブなデータバインディングとコンポーネントシステムによりUI構築を効率化できる。Composition APIはロジックの再利用性と整理に優れている |
| **ビルドツール** | Vite | HMRによる高速な開発サーバーを提供し、開発体験を大幅に向上させるため |
| **ルーティング** | Vue Router | Vue.js公式のルーティングライブラリであり、SPAにおけるページ遷移管理のデファクトスタンダード |
| **HTTPクライアント** | Axios | PromiseベースのAPIで非同期通信を容易に扱える。リクエストインターセプター機能により認証トークンの自動付与を効率的に実装できる |
| **リアルタイム通信** | socket.io-client | バックエンドの `Flask-SocketIO` と対になる公式クライアントライブラリ。WebSocket通信を安定して実装できる |
| **状態管理** | Vue `reactive` API | グローバルな状態（認証情報など）が限定的なため、PiniaやVuexは導入せずVue標準機能でシンプルに実装する |
| **グラフ描画** | Chart.js / vue-chartjs | 豊富なグラフ種類とカスタマイズ性を持ち、Vueコンポーネントとして容易に統合できる |

## 3. UI/UX設計方針

### 3.1. レスポンシブデザイン

- **方針**: CSSのメディアクエリ（`@media`）を積極的に使用し、PC・タブレット・スマートフォンなどあらゆるデバイスサイズで最適な表示と操作性を提供する。
- **実装**:
  - ブレークポイントは `768px` を基準とし、これより小さい画面ではモバイル用レイアウトを適用する。
  - モバイルレイアウトでは複数カラムのグリッドを1カラムに変更し、文字サイズや余白を適切に縮小してコンテンツの可読性と操作性を確保する。

### 3.2. ダークモード対応

- **方針**: ユーザーのOSやブラウザのダークモード設定に左右されず、常に意図したデザインで表示されるようにする。
- **実装**: 各ページのルート要素（例: `.dashboard-container`）に対して、明示的に背景色（`background-color`）と基本の文字色（`color`）を指定する。これにより、ダークモードが有効な環境でも背景と文字のコントラストが維持され、視認性が損なわれない。

## 5. 画面（ビュー）詳細設計

### 5.1. `LoginView.vue`

- **役割**: ユーザー認証の入り口。
- **主要なUI要素**: メールアドレス入力欄、パスワード入力欄、ログインボタン、エラーメッセージ表示エリア。
- **管理する状態**: `email`, `password`, `isLoading`, `errorMessage`。
- **主要なロジック**: `handleLogin` メソッド内で `api/auth.js` の `login` 関数を呼び出す。成功時には返されたアクセストークンを `store/auth.js` に保存し、`vue-router` を使ってダッシュボード（`/`）に遷移する。
- **スタイル**: 画面全体を覆うコンテナに明示的な背景色と文字色を指定し、ダークモードの影響を防ぐ。

### 5.2. `DashboardView.vue`

- **役割**: ログイン後のメイン画面。ユーザーが所有する全デバイスを一覧表示・管理する。
- **主要なUI要素**: ヘッダー（ログアウトボタン・統計ページへのリンク）、新規機器登録ボタン、デバイスカードのグリッド。
- **管理する状態**: `devices`（APIから取得したデバイスリスト）、`isLoading`、`errorMessage`、`isModalOpen`（登録モーダルの表示状態）、`isEditModalOpen`（編集モーダルの表示状態）。
- **ライフサイクル**:
  - `onMounted`: `api/devices.js` の `getDevices` を呼び出しデバイスリストを初期化する。`socketService` に接続し、`update:status` イベントを購読して各デバイスのオンライン状態をリアルタイムで更新する。
  - `onUnmounted`: ページを離れる際に `socketService` から切断する。
- **主要なロジック**:
  - `handleNewDevice`: 登録モーダルから渡された新しいデバイス情報を `devices` リストに追加する。
  - `openEditModal`: 編集ボタンが押されたデバイスの情報を `selectedDeviceForEdit` にセットし、編集モーダルを開く。
  - `handleDeleteDevice`: 削除ボタンが押されたデバイスのIDを `api/devices.js` の `deleteDevice` に渡す。成功時にはリストから該当デバイスを即座に削除しUIに反映させる。
- **スタイル**: メディアクエリ（`max-width: 768px`）を適用。画面が狭い場合、カード内の余白と文字サイズを縮小し、ヘッダーの要素が縦に並ぶようにレイアウトを変更する。

### 5.3. `DeviceDetailView.vue`

- **役割**: 単一デバイスのリアルタイム監視と、遠隔操作・設定を行う。
- **主要なUI要素**: カウンター表示エリア、遠隔操作ボタン、カメラ映像表示エリア、ROI編集用キャンバス、ROI編集コントロール。
- **管理する状態**: `device`（APIから取得した初期データ）、`isOnline`、`todayCount`、`totalCount`（Socket.IOで更新）、`streamSrc`（映像データ）、ROI編集関連の各種フラグ。
- **ライフサイクル**:
  - `onMounted`:
    1. `vue-router` からURLパラメータ（`deviceId`）を取得する。
    2. `axios` を直接使用して `/api/devices/{deviceId}` を呼び出す。
    3. `socketService` でサーバーに接続し、`deviceId` をペイロードに含めて `browser:join_room` イベントを送信する。
    4. `update:stats`、`stream:frame`、`update:status` イベントのリスナーを登録する。
  - `onUnmounted`: `socketService.disconnect()` を呼び出しサーバーとの接続をクリーンに切断する。
- **主要なロジック**:
  - `triggerPunch`: 「正拳発動！」ボタン押下時に `socketService` 経由で `command:execute_action` イベントをサーバーに送信する。
  - **ROI編集**:
    1. 編集モードでは `canvas` 上でマウスイベントを検知し、ユーザーが描画した矩形を `rois`（表示座標系）に一時保存する。
    2. 保存ボタン押下時、表示画像のサイズとカメラのネイティブ解像度（640×480）の比率を計算し、**表示座標系をネイティブ座標系に変換**してからサーバーに送信する。
    3. サーバーから取得したROIデータ（ネイティブ座標系）を描画する際は、逆の計算で**ネイティブ座標系を表示座標系に変換**してから `canvas` に描画する。
- **スタイル**: メディアクエリ（`max-width: 768px`）を適用。画面が狭い場合、2列グリッドを1列に変更し、統計情報の文字サイズをモバイル向けに調整する。

### 5.4. `StatisticsView.vue`

- **役割**: 複数デバイスの活動履歴をグラフで比較・可視化する。
- **主要なUI要素**: 戻るリンク、ヘッダー、機器選択チェックボックス群、期間選択ボタン（月/年）、グラフ描画エリア。
- **管理する状態**: `devices`（全デバイスリスト）、`selectedDeviceIds`（チェックされたデバイスIDのリスト）、`period`、`chartData`（Chart.js用データ）、`isLoading`、`errorMessage`。
- **主要なロジック**:
  1. `onMounted` で全デバイスリストを取得し、チェックボックスを生成する。
  2. `watch` エフェクトを使い、`selectedDeviceIds` または `period` が変更されるたびに `api/devices.js` の `getComparisonStats` を呼び出す。
  3. サーバーから返された、デバイスIDをキーとするデータオブジェクトを受け取る。
  4. 全データセットに共通する日付ラベルのリストを生成する。
  5. 各デバイスのデータを `vue-chartjs` が要求する複数データセットの形式に整形し、`chartData` を更新してグラフを再描画する。

## 6. API連携と状態管理

### 6.1. REST API（`axios`）

`src/api/axios.js` にて `axios` の共通インスタンスを生成。**リクエストインターセプター**を使い、全リクエストの `Authorization` ヘッダーに `localStorage` から読み取ったJWTアクセストークンを `Bearer <token>` 形式で自動付与する。

### 6.2. リアルタイムAPI（`socket.io-client`）

`src/api/socketService.js` にて Socket.IO の接続を管理するシングルトンサービスを実装。接続時には認証トークンをサーバーに送信する。コンポーネントはこのサービスを介してイベントの送受信を行う。

### 6.3. ルーティング（`vue-router`）

`src/router/index.js` にてページのパスとコンポーネントを定義。**ナビゲーションガード（`router.beforeEach`）**を実装し、遷移先のルートが `meta: { requiresAuth: true }` を持つ場合、`store/auth.js` の認証状態をチェックする。未認証の場合は `/login` ページに強制リダイレクトする。

### 6.4. 状態管理（`src/store/auth.js`）

Vue 3の `reactive` APIを利用し、`token` と `isAuthenticated` フラグを管理。`localStorage` と同期することで、ページリロード後もログイン状態を維持する。
