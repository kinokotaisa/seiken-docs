# わかば正拳突き人形 仕様書

## 概要

スライダクランク機構とサーボモータを組み合わせた、正拳突き動作を行う電動フィギュア。  
圧力センサによるボール検知、または手動ボタン操作により、連続ワンツーパンチ3回 + 音声出力を行うスタンドアロン動作が基本構成。  
Raspberry Piと接続することで、seikenシステム（遠隔操作Webシステム）からネットワーク経由で正拳をトリガーする拡張構成にも対応する。

---

## システム構成

### スタンドアロン構成

```
[電源: 単3×3本 4.5V]
        │
        ▼
   [ATtiny402] ──── [サーボモータ R] ──── [スライダクランク機構]
        │      └─── [サーボモータ L] ──── [スライダクランク機構]
        │
        ├──── [圧力センサ FSR402/406] ← ボールを検知
        ├──── [マニュアルパンチボタン] ← 手動トリガー
        └──── [DFPlayer Mini] ──── [スピーカー 8Ω]
```

### seikenシステム連携構成

Raspberry Piと接続することで、遠隔操作システム（seikenシステム）からの制御が可能になる。

```
[Webブラウザ] ──Socket.IO──► [Webサーバー] ──Socket.IO──► [Raspberry Pi]
                                                                  │
                                                           GPIO出力（3.3V）
                                                                  │
                                                               [1kΩ] ← 保護抵抗
                                                                  │
[電源: 単3×3本 4.5V]                                             │
        │                                                         │
      [10kΩ] プルアップ ──────────────────────────── PA2（物理5ピン）
        │                                                         │
   [ATtiny402] ──── [サーボモータ R / L] ──── [スライダクランク機構]
        └──── [DFPlayer Mini] ──── [スピーカー 8Ω]
```

ラズパイのGPIOからATtiny402の物理5ピン（PA2）に接続し、LOWパルスを送ることで正拳をトリガーする。圧力センサと同じピンを共用しているため、外部トリガーとして機能する。

---

## 動作仕様

| トリガー | 動作 |
| :---- | :---- |
| 圧力センサ検知（val < 300） | 音声再生 → 連続ワンツーパンチ × 3回 |
| マニュアルパンチボタン押下 | 音声再生 → 連続ワンツーパンチ × 3回 |

### サーボ角度定義

| 定数 | 値 | 意味 |
| :---- | :---- | :---- |
| `Angle1` | 150° | 引き（初期位置） |
| `Angle2` | 30°  | 突き出し位置 |

### ワンツーシーケンス（1セット = 約2.4秒）

```
servoR: Angle2(突き) → Angle1(戻し)
servoL: Angle1(突き) → Angle2(戻し)
各ステップ delay 300ms
```

### ATtiny402 ピンアサイン

| 物理ピン | プログラム番号 | ポート | 用途 |
| :---- | :---- | :---- | :---- |
| 1 | — | VDD | +4.5V電源入力（電源スイッチ経由） |
| 2 | 0 | PA6 | サーボ出力 L |
| 3 | 1 | PA7 | サーボ出力 R |
| 4 | 2 | PA1 | DFPlayer Mini ADKEY1（再生トリガー） |
| 5 | 3 | PA2 | 圧力センサ入力 `analogRead()` / 10kΩ プルアップ |
| 6 | 7 | PA0 | UPDI書き込み端子（J3ソケット接続） |
| 7 | 4 | PA3 | マニュアルパンチボタン入力 / 10kΩ プルアップ |
| 8 | — | GND | GND |

> **注意**: コード内のピン番号指定は物理ピン番号ではなく「プログラム番号」を使用すること。

### 圧力センサ回路

プルアップ構成。センサに負荷がかかると抵抗値が下がり、ADC読み値が低下する。

```
VDD (4.5V)
 │
[10kΩ]  ← プルアップ抵抗
 │
 ├──── PA2 (analogRead)
 │
[FSR402/406]
 │
GND
```

| 状態 | FSR抵抗 | ADC読み値（目安） |
| :---- | :---- | :---- |
| 無負荷 | 1MΩ | ≈ 1023 |
| ボール検知 | 〜数kΩ | < 300 |
| 全負荷 | 1kΩ | ≈ 0 |

### ラズパイ接続時の保護回路

seikenシステム連携時、ラズパイのGPIO（3.3V）とATtinyのプルアップ電源（4.5V）が同じノードに接続される。電圧差による意図しない電流を防ぐため、ラズパイ〜ATtiny間に**1kΩの保護抵抗**を直列に挿入すること。

```
VDD (4.5V)
 │
[10kΩ]  ← 既存のプルアップ抵抗
 │
 ├──── PA2 (ATtiny402 物理5ピン)
 │
[1kΩ]   ← 保護抵抗（追加）
 │
ラズパイ GPIO (3.3V HIGH / 0V LOW)
```

| ラズパイ GPIO | PA2電圧（目安） | ATtiny動作 |
| :---- | :---- | :---- |
| LOW (0V) | ≈ 0V | val < 300 → 正拳発動 |
| HIGH (3.3V) | ≈ 3.3V | val ≈ 1023 → 待機 |
| 未接続 | ≈ 4.5V | val ≈ 1023 → 待機 |

> **注意**: 保護抵抗なしで直結すると、GPIO HIGH時にプルアップ電源（4.5V）との電圧差（1.2V）により、VDDからGPIOピンへ向かって電流が逆流する（約0.12mA）。通常動作時の電流値自体は小さく即座に破損するわけではないが、電源投入タイミングのズレ・GPIOの出力モード切り替え瞬間・ノイズや静電気など過渡的な状態では瞬間的に大きな電流スパイクが発生する可能性がある。1kΩを直列挿入することでこの電流スパイクを制限し、ラズパイGPIOおよびATtiny402への長期的なダメージリスクを低減できる。必ず1kΩを挿入すること。

### DFPlayer Mini トリガー方式

ADKEY1端子をATtiny402から直接制御。  
再生時のみ出力モードに切り替え、待機時は入力（高インピーダンス）に戻してADKEY1回路への影響を最小化する。

```
再生シーケンス：
pinMode(OUTPUT) → digitalWrite(LOW) → delay(150ms) → digitalWrite(HIGH) → pinMode(INPUT)
```

## ファームウェア仕様

### 開発環境

| 項目 | 内容 |
| :---- | :---- |
| IDE | Arduino IDE |
| ボードパッケージ | megaTinyCore |
| 書き込み方式 | UPDI（SerialUPDI または JTAG2UPDI） |
| ライブラリ | `Servo.h`（megaTinyCore付属） |

### ソースコード（最終版）

```cpp
/* ATtiny402用プログラム（圧力センサ版）*/
#include <Servo.h>

Servo servoR;
Servo servoL;

const int Angle1 = 150; // 引き位置（初期位置）
const int Angle2 = 30;  // 突き出し位置

const int PS  = 3; // 圧力センサ（物理5ピン / PA2）
const int MPB = 4; // マニュアルパンチボタン（物理7ピン / PA3）
const int DF  = 2; // DFPlayer Mini ADKEY1（物理4ピン / PA1）

int val     = 0; // 圧力センサADC値
int val_MPB = 0; // ボタン入力値
int i       = 0; // 正拳回数カウント

void setup() {
  servoR.attach(1); // PA7
  servoL.attach(0); // PA6
  pinMode(MPB, INPUT); // 外部プルアップのためINPUT
  pinMode(DF,  INPUT);
  servoR.write(Angle2);
  servoL.write(Angle1);
}

void loop() {
  // 初期位置に戻す
  servoR.write(Angle1);
  servoL.write(Angle2);

  val_MPB = digitalRead(MPB);
  val     = analogRead(PS); // プルアップ構成：無負荷≈1023、検知時<300

  // 圧力センサ検知（val < 300）またはボタン押下（LOW）で正拳発動
  if (val < 300 || val_MPB == LOW) {

    // DFPlayer再生トリガー
    pinMode(DF, OUTPUT);
    digitalWrite(DF, LOW);
    delay(150);
    digitalWrite(DF, HIGH);
    pinMode(DF, INPUT); // 高インピーダンスに戻す

    // 正拳突き3回（ワンツー × 3セット）
    for (i = 0; i < 3; i++) {
      servoR.write(Angle2); delay(300);
      servoR.write(Angle1); delay(300);
      servoL.write(Angle1); delay(300);
      servoL.write(Angle2); delay(300);
    }
  }
}
```

| 識別子 | 型 | 値 | 説明 |
| :---- | :---- | :---- | :---- |
| `Angle1` | `const int` | 150 | サーボ引き位置（°） |
| `Angle2` | `const int` | 30 | サーボ突き位置（°） |
| `PS` | `const int` | 3 | 圧力センサ入力ピン番号 |
| `MPB` | `const int` | 4 | マニュアルパンチボタンピン番号 |

| `val_MPB` | `int` | — | ボタン入力値（HIGH/LOW） |
| `i` | `int` | — | 正拳ループカウンタ |

## 開発履歴

### プロトタイプ1（ATtiny202使用）

マイコンをATtiny202で試作。LEDで圧力センサ検知を可視化する構成。  
最終版（ATtiny402）への移行時に以下の問題を修正・改善した。

| 問題 | 対応 |
| :---- | :---- |
| `INPUT_PULLUP` が効かない | 外部プルアップ抵抗（10kΩ）に変更 |
| ATtiny202のFlash容量不足 | ATtiny402（Flash 4KB）に変更 |
| 正拳アームが外れそう | 接合部の設計見直し（要対応） |
| ケーブルが長すぎる | 配線整理（要対応） |
| Bambu PLA MATTEの摩擦が大きい | 動作確認済み・要継続観察 |

### 最終版（ATtiny402使用）

- DFPlayer Miniによる音声出力を追加
- 圧力センサをプルアップ構成で安定動作
- ADKEY1制御をpinMode切り替え方式で実装
